# 教务处端口问题修复计划

## 问题分析

### 1. 教师二级窗口数据关联问题
- **问题描述**：所有教师的二级窗口显示相同的统计数据，班级数、学生数、词表数、正确率没有与具体老师关联
- **根本原因**：`db.getStudentsByTeacher` 方法实现有误，当学生没有 `teacherId` 时会返回所有学生，而不是只返回该教师的学生
- **影响范围**：所有教师的统计数据显示

### 2. 学生二级窗口教师姓名关联问题
- **问题描述**：学生二级窗口没有显示教师姓名
- **根本原因**：学生数据中可能缺少 `teacherId` 字段，或者 `db.findTeacher` 方法无法正确找到教师
- **影响范围**：所有学生的教师信息显示

### 3. 学生积分同步问题
- **问题描述**：学生的学习积分和记录没有同步到后端数据库，API 返回 500 错误
- **根本原因**：`api.updateStudentCoins` 方法使用了 `POST` 方法，但后端可能期望 `PUT` 方法
- **影响范围**：学生积分同步功能

## 修复计划

### 任务 1：修复教师数据关联问题
- **目标**：确保每个教师只看到自己的学生、词表和学习记录
- **步骤**：
  1. 修改 `db.getStudentsByTeacher` 方法，确保只返回与教师关联的学生
  2. 确保 `db.getWordListsByTeacher` 和 `db.getLearningLogsByTeacher` 方法正确实现
  3. 测试教师二级窗口数据显示

### 任务 2：修复学生教师关联问题
- **目标**：确保学生二级窗口正确显示教师姓名
- **步骤**：
  1. 检查学生数据结构，确保每个学生都有 `teacherId` 字段
  2. 确保 `db.findTeacher` 方法能正确根据 `teacherId` 找到教师
  3. 测试学生二级窗口教师姓名显示

### 任务 3：修复学生积分同步问题
- **目标**：确保学生积分能正确同步到后端数据库
- **步骤**：
  1. 修改 `api.updateStudentCoins` 方法，将 HTTP 方法从 `POST` 改为 `PUT`
  2. 测试学生积分同步功能
  3. 确保学习记录也能正确同步

## 实施时间估计
- 任务 1：30 分钟
- 任务 2：20 分钟
- 任务 3：20 分钟
- 总计：70 分钟

- 测试所有学生的二级窗口教师姓名显示

- 确保所有功能正常运行
